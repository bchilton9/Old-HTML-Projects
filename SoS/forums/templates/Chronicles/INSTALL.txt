## Comments:		There are a number of minor problems, they're all just visual glitches
##			and there was no way of getting rid of those without having to
##			release this template as a MOD. You're free to fix those problems if
##			find it worth the time. Be aware that it involves editing the actual
##			phpBB2 code, so don't do this unless you know at least something about
##			PHP programming.
##			One of the problems is that some links are hard to distinguish due to badly
##			matching font colors and backgrounds with low contrast.
##			I won't offer any instructions on how to do it and which files to edit
##			and so on. Generally, these problems can be fixed by adding a class
##			attribute to the HTML tag that is generated by phpBB. Then create a
##			corresponding class in the style sheet file (Chronicles.css) and
##			control its visual appearance by editing that class.
##			If you haven't got the faintest idea about what I mean, go look for a
## 			CSS "get started" guide or some tutorials on the subject. Webmonkey
##			(http://www.webmonkey.com) is a good place to look for such things.

# Ready to go? Ok, Let's go!
#
#
#----[ UPLOAD (or COPY if you run phpBB on localhost) all of the files ]------
#----[ and folders to the templates directory of your phpBB directory ]-------
#
#----[ OPEN THE ADMIN PANEL ]-------------------------------------------------
#----[ CLICK Styles Admin -> Add ]--------------------------------------------
#
#    If you've successfully completed the previous steps, Chronicles should be
#    in the list. If it's not, redo the previous step and make sure all of the
#    Chronicles folders and files are located in the template directory of your
#    phpBB2 root directory. The Chronicles folder should contain a number of
#    .tpl, .cfg and .css files (see the file list above).
# 
#
#    If Chronicles is in the list, click "Install" to install it. The template
#    settings will now be inserted into your database.
#
#    When you've installed the template you may want to set it as default or make it
#    override the user's choice of template (not recommended). You can do so by
#    in the admin panel clicking General Admin -> Configuration.
